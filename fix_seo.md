أنت Senior Django SEO Engineer + Technical SEO Specialist. 
مطلوب منك إصلاح مشاكل SEO وTechnical SEO الموجودة في المشروع الحالي بشكل عملي ومباشر داخل الكود، وليس مجرد تحليل أو شرح.

الدومين الأساسي للموقع هو:
https://elwsamshop.com

المطلوب:
نفّذ الإصلاحات داخل المشروع الحالي مع الحفاظ على عدم كسر أي وظيفة موجودة، واعتبر أن النسخة الأساسية الوحيدة الصحيحة للموقع هي:
https://elwsamshop.com

هذه هي المشاكل المؤكدة التي يجب إصلاحها:

1) توحيد نسخة الدومين
- حاليًا:
  - http://elwsamshop.com
  - https://www.elwsamshop.com
  يعملان 200 بدل التحويل 301 إلى النسخة الأساسية.
- المطلوب:
  - فرض 301 من http -> https
  - فرض 301 من www -> non-www
  - التأكد أن كل canonical يشير فقط إلى https://elwsamshop.com
  - لا تترك أي نسخة بديلة تعمل كنسخة مستقلة

2) إصلاح رابط معطوب داخل sitemap.xml
- يوجد رابط منتج يحتوي %5C ويدخل في infinite redirect loop.
- المطلوب:
  - اكتشاف سبب تكوين الـ slug أو الرابط الخاطئ
  - إصلاحه من المصدر داخل الموديل / الفيو / التوليد
  - تنظيف أو تصحيح أي slug معطوب
  - التأكد أن روابط الـ sitemap كلها ترجع:
    - 200 مباشرة
    أو
    - 301 واحد فقط إلى canonical صحيح
  - منع تكرار المشكلة مستقبلًا عند توليد sitemap

3) إصلاح صفحة البحث
- صفحة /search/?q=... ترجع 500
- المطلوب:
  - إصلاح صفحة البحث بالكامل
  - التأكد أن البحث يعمل بدون أخطاء
  - لو كانت هناك مشكلة جوهرية تمنع إصلاحه بسرعة، احذف SearchAction من structured data مؤقتًا بدل تركه مكسورًا
  - لو تم إصلاح البحث، اجعل الـ schema والـ robots متوافقين معه بشكل صحيح

4) حل التناقض بين SearchAction و robots.txt
- الـ structured data يشير إلى /search/
- بينما robots.txt يمنع /search/
- والصفحة أصلًا معطوبة
- المطلوب:
  - توحيد السلوك
  - إمّا:
    أ) تفعيل صفحة البحث بشكل صحيح والسماح بها إذا كان هذا هو الأفضل
    أو
    ب) إزالة SearchAction نهائيًا إذا لم تكن الصفحة جاهزة
- اختر الحل الأفضل SEO وتقنيًا ونفّذه

5) تحسين الكاش وتقليل الـ cookies على الصفحات العامة
- الصفحات العامة ترسل csrftoken و Vary: Cookie حتى للمستخدم غير المسجل
- هذا يضعف caching ويرفع TTFB
- المطلوب:
  - تقليل إنشاء الـ cookies غير الضرورية للزوار غير المسجلين
  - تقليل Vary: Cookie على الصفحات العامة قدر الإمكان
  - تحسين قابلية cache للـ anonymous pages
  - الحفاظ على الأمان وعدم كسر الـ forms أو CSRF حيث يلزم فعليًا

6) تقليل حجم HTML في صفحات التصنيفات
- صفحة التصنيف ثقيلة جدًا، والـ HTML وحده كبير جدًا
- المطلوب:
  - تقليل حجم الـ HTML الناتج
  - تقليل أي inline CSS/JS كبير
  - نقل ما يمكن إلى ملفات خارجية
  - تقليل أي structured data مبالغ فيه أو ItemList ضخم جدًا
  - الحفاظ على UX والتصميم الحالي قدر الإمكان

7) ضغط وتحسين الصور الأساسية
- الصور الثقيلة مثل:
  - slide1.png
  - ELWSAM-LOGO2020.png
  تؤثر على الأداء وLCP
- المطلوب:
  - تحويل الصور المناسبة إلى WebP أو AVIF إن أمكن
  - تقليل الأبعاد والحجم
  - الحفاظ على جودة مناسبة
  - preload فقط للصورة الحرجة فوق الطية الأولى
  - بقية الصور تكون محسنة بشكل مناسب

8) lazy loading للصور غير الحرجة
- الصفحة الرئيسية فيها صور بدون loading="lazy"
- المطلوب:
  - إضافة lazy loading لكل الصور غير الحرجة
  - استثناء صور الـ hero أو أي عنصر LCP أساسي فقط

9) تحسين محتوى صفحات المنتجات
- محتوى كثير من المنتجات ضعيف جدًا ومختصر
- المطلوب:
  - تحسين بنية صفحة المنتج بحيث تدعم محتوى SEO أفضل
  - تجهيز قالب مناسب يسمح بوصف عملي مفيد
  - إضافة أو دعم أقسام مثل:
    - وصف واضح
    - الاستخدامات
    - المميزات
    - المواصفات
    - FAQ
  - لا تكتب كلام حشو أو مكرر
  - اجعل المحتوى يخدم نية الباحث والشراء

10) تحسين الصفحة الرئيسية
- النص الحالي عام جدًا ولا يستهدف كلمات البحث التجارية بشكل كفاية
- المطلوب:
  - إعادة كتابة الـ hero وintro لتكون أقوى SEO وتجاريًا
  - مراعاة كلمات مثل:
    - مشترك كهرباء
    - برايز
    - كشاف ليد
    - وأي كلمات مرتبطة فعلًا بمنتجات المتجر
  - تحسين الربط الداخلي من الصفحة الرئيسية

11) استغلال المنتجات المعروضة في الهوم
- توجد featured_products و latest_products في الـ view لكن الصفحة لا تستفيد منها بالشكل المطلوب
- المطلوب:
  - إضافة بلوكات واضحة داخل الهوم مثل:
    - الأكثر طلبًا
    - أحدث المنتجات
  - مع روابط مباشرة للمنتجات
  - وتحسين internal linking

12) تحسين صفحات التصنيفات
- العناوين والـ meta descriptions قالبية ومختصرة
- المطلوب:
  - جعل title و meta description مخصصين لكل تصنيف
  - إضافة intro نصي SEO مناسب لكل قسم
  - تحسين قابلية التوسع مستقبلًا

13) إصلاح breadcrumb schema
- breadcrumb schema في صفحة المنتج يضع "الرئيسية" بدون item URL
- المطلوب:
  - جعل breadcrumb schema مكتملًا وصحيحًا

14) تنظيف Product schema
- يوجد خصائص غير معيارية مثل:
  - size_length
  - urlSlug
- المطلوب:
  - إزالة أي خصائص غير مدعومة أو غير قياسية
  - الالتزام فقط بما يفيد Google Product Rich Results و schema.org بشكل صحيح

15) إزالة خطر templates - Copy
- يوجد مجلدان:
  - templates
  - templates - Copy
- ويبدو أن هناك خطر أن المشروع يقرأ من مكان غير المقصود
- المطلوب:
  - مراجعة إعدادات TEMPLATES['DIRS']
  - التأكد أن المشروع يستخدم المسار الصحيح فقط
  - إزالة أي التباس قد يسبب تعديل ملفات غير مستخدمة

16) إصلاح روابط السوشيال المعطلة
- بعض الروابط ما زالت #
- المطلوب:
  - استبدالها بروابط صحيحة لو متوفرة
  - أو إخفاؤها/إزالتها مؤقتًا بدل تركها معطلة

مراجع داخل المشروع يجب مراجعتها أولًا:
- project/settings.py
- core/context_processors/seo.py
- products/views.py
- products/models.py
- templates
- templates - Copy
- ملفات sitemap
- ملفات robots
- قوالب الصفحة الرئيسية
- قوالب صفحة المنتج
- قوالب صفحة التصنيف
- أي middlewares أو context processors أو SEO helpers ذات صلة

تعليمات التنفيذ:
- لا تكتفِ بالشرح، نفّذ التعديلات فعليًا في الكود
- لا تقترح فقط، بل عدّل الملفات المطلوبة
- لا تكسر الوظائف الحالية
- حافظ على أفضل ممارسات Django وSEO
- استخدم حلول Production-ready
- لا تضف hacks مؤقتة إلا عند الضرورة، ولو استخدمتها اذكرها بوضوح
- لا تترك أي TODO بدون شرح
- لا تحذف أي وظيفة مهمة إلا لو كانت مكسورة ومؤذية SEO، وفي هذه الحالة اذكر السبب

3) اعرض لي التعديلات على شكل واضح:
   - اسم الملف
   - ماذا عدلت
   - لماذا عدلته
4) اعرض patch / diff أو الكود النهائي لكل جزء تم تغييره
5) اعرض Checklist تحقق نهائية تشمل:
   - هل كل نسخ الدومين تتحول 301؟
   - هل sitemap نظيفة؟
   - هل /search/ تعمل؟
   - هل schema صحيح؟
   - هل robots متوافق؟
   - هل تم تقليل cookies والكاش أفضل؟
   - هل تم تقليل HTML؟
   - هل الصور تحسنت؟
   - هل lazy loading تم؟
   - هل الهوم والتصنيفات والمنتجات تحسنت SEO؟
6) في النهاية اعطني:
   - Summary مختصر

   - خطوات يدوية مطلوبة على السيرفر أو CDN أو Nginx/Apache إن وجدت

- صفحة البحث لا تعطي 500
- لا يوجد structured data متناقض مع robots أو الواقع
- الصفحة الرئيسية والتصنيفات والمنتجات أفضل SEO وتقنيًا
- الأداء أفضل بدون كسر الموقع
- الكود أوضح وأنظف من السابق
